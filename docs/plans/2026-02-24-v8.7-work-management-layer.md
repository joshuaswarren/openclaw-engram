# v8.7 Work Management Layer (Tasks/Projects/Kanban) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add optional work-management primitives (tasks, projects, and board views) to support agent execution workflows directly in Engram.

**Architecture:** Create a separate, optional workspace layer for work items that can interoperate with memory extraction but remains logically distinct from durable personal memory facts.

**Tech Stack:** TypeScript, `src/work/*`, `src/cli.ts`, `src/tools.ts`, markdown frontmatter files, node:test.

---

## Requirements

1. Task CRUD with status transitions and ownership metadata.
2. Project records with task linkage.
3. Board exports (kanban-style markdown) for visual workflow tools.
4. Keep work-layer data out of default long-term memory extraction unless explicitly linked.
